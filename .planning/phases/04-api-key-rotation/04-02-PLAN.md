**SQLiteApiKeyStore.rotate_key(key_id, grace_period_seconds=86400, rotated_by="system") -> str:**

1. Look up existing key by key_id (must be non-revoked, non-rotated)
2. Generate new raw key + hash + key_id
3. In a transaction: INSERT new key row with same principal/tenant/groups, UPDATE old key row: set rotated_to_key_id = new_key_id, grace_until = now + grace_period
4. Emit KeyRotated event via _event_publisher
5. Return new raw key

**SQLiteApiKeyStore.get_principal_for_key() changes:**
After fetching row, check:

- If rotated_to_key_id is not NULL (key has been rotated):
- If now < grace_until -> return principal normally (grace active)
- If now >= grace_until -> raise ExpiredCredentialsError("API key has been rotated and grace period has expired")

**PostgresApiKeyStore.rotate_key():** Same logic as SQLite but using %s params and RETURNING clause.

**PostgresApiKeyStore.get_principal_for_key():** Same grace period logic.

Import KeyRotated from domain.events in both files. Follow the established event publishing pattern (check self._event_publisher before calling).

**Error handling:**

- Key not found -> return False
- Key revoked -> raise ValueError("Cannot rotate revoked key")
- Key already rotated (rotated_to_key_id not null and grace_until > now) -> raise ValueError("Key already has pending rotation")

```python
def rotate(self, new_key_id: str, grace_until: datetime, rotated_by: str) -> None:
    if self._revoked:
        raise ValueError(f"Cannot rotate revoked key {self._key_id}")
    if self._rotated_to_key_id is not None and self._grace_until and datetime.now(UTC) < self._grace_until:
        raise ValueError(f"Key {self._key_id} already has pending rotation")
    self._record_event(KeyRotated(...))
    self._rotated_to_key_id = new_key_id
    self._grace_until = grace_until
```
