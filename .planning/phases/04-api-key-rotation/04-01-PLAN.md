  <behavior>
API key rotation behavior:

**rotate_key(key_id, grace_period_seconds=86400, rotated_by="system") -> str (new raw key):**

- Finds the existing key by key_id
- Generates a new raw API key + hash
- Stores the new key entry with same principal/metadata as old key
- Marks the old key entry with a grace_until timestamp (now + grace_period_seconds)
- Emits KeyRotated domain event
- Returns the new raw key (shown only once)

**get_principal_for_key(key_hash) behavior changes:**

- If key_hash matches an active (non-rotated, non-revoked, non-expired) key -> return principal (no change)
- If key_hash matches a rotated key AND now < grace_until -> return principal (grace period active)
- If key_hash matches a rotated key AND now >= grace_until -> raise ExpiredCredentialsError("API key has been rotated and grace period has expired")
- If key_hash not found -> return None (no change)

**KeyRotated domain event fields:**

- key_id: str (the key_id that was rotated)
- principal_id: str
- new_key_id: str (the new key's key_id)
- rotated_at: float (timestamp)
- grace_until: float (timestamp when old key becomes invalid)
- rotated_by: str

**Error cases:**

- rotate_key with unknown key_id -> return False or raise ValueError
- rotate_key on already-revoked key -> raise ValueError("Cannot rotate revoked key")
- rotate_key on already-rotated key that is in grace period -> raise ValueError("Key already has pending rotation")

Implementation order:

**Step 1.** Add KeyRotated domain event to domain/events.py:

```python
@dataclass
class KeyRotated(DomainEvent):
    key_id: str
    principal_id: str
    new_key_id: str
    rotated_at: float
    grace_until: float
    rotated_by: str

    def __post_init__(self):
        super().__init__()
```

**Step 2.** Add rotate_key to IApiKeyStore protocol in domain/contracts/authentication.py:

```python
@abstractmethod
def rotate_key(
    self,
    key_id: str,
    grace_period_seconds: float = 86400,
    rotated_by: str = "system",
) -> str:
    """Rotate an API key, producing a new key while keeping old valid during grace period."""
    ...
```

**Step 3.** Implement on InMemoryApiKeyStore in api_key_authenticator.py:

- Add a _rotated_keys dict: old_key_hash -> (new_key_hash, grace_until)
- In rotate_key(): find key by key_id, generate new key, store mapping, emit event
- In get_principal_for_key(): after main lookup fails, check_rotated_keys for grace period

Design choice: Track rotation state on the ApiKeyMetadata by adding optional rotated_to_key_id and grace_until fields to the metadata dict. This keeps ApiKeyMetadata dataclass unchanged (use metadata dict for extension). The _rotated_keys dict provides fast O(1) lookup of old_hash -> grace_until.

**Step 4.** Grace period enforcement in get_principal_for_key():
The constant-time lookup already iterates all keys. After finding a match:

- Check if this key_hash exists in_rotated_keys
- If yes, check if grace period is still active (now < grace_until)
- If grace expired, raise ExpiredCredentialsError with clear message about rotation
- If grace active, return principal normally

**Step 5.** Event publishing: Follow established pattern from Phase 2 (optional event_publisher callback):
The InMemoryApiKeyStore currently does NOT have event_publisher. Add it as optional parameter to **init** with backward compatibility (default None). Use _publish_event helper pattern from AuthRateLimiter.
