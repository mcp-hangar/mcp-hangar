        pass

        pass

  <files>enterprise/behavioral/profiler.py, enterprise/behavioral/bootstrap.py, enterprise/behavioral/__init__.py</files>
  <behavior>
    - BehavioralProfiler(baseline_store, config) satisfies IBehavioralProfiler Protocol
    - BehavioralProfiler.get_mode delegates to baseline_store.get_mode
    - BehavioralProfiler.set_mode delegates to baseline_store.set_mode and logs the transition
    - BehavioralProfiler.record_observation delegates to baseline_store.record_observation ONLY when provider mode is LEARNING; no-op in DISABLED and ENFORCING modes
    - bootstrap_behavioral(db_path, config) returns a configured BehavioralProfiler instance
    - bootstrap_behavioral logs "behavioral_profiling_initialized" at INFO level
  </behavior>
  <action>
    1. Create `enterprise/behavioral/profiler.py`:
       - BSL 1.1 docstring.
       - Import `structlog`, `BehavioralMode`, `NetworkObservation` from `mcp_hangar.domain.value_objects.behavioral`.
       - Import `IBaselineStore` from `mcp_hangar.domain.contracts.behavioral` (for type hint only; no runtime dependency on the interface).
       - Class `BehavioralProfiler`:
         - `__init__(self, baseline_store: "IBaselineStore", config: dict | None = None)`: Store `_baseline_store` and `_config`. Create structlog logger.
         - `get_mode(self, provider_id: str) -> BehavioralMode`: Delegate to `self._baseline_store.get_mode(provider_id)`.
         - `set_mode(self, provider_id: str, mode: BehavioralMode) -> None`:
           - Get `learning_duration_hours` from config (default 72).
           - Delegate to `self._baseline_store.set_mode(provider_id, mode, learning_duration_hours)`.
           - Log `"behavioral_mode_changed"` with provider_id and new mode.
         - `record_observation(self, observation: NetworkObservation) -> None`:
           - Get mode for `observation.provider_id`.
           - If mode is `LEARNING`: delegate to `self._baseline_store.record_observation(observation)`.
           - If mode is `DISABLED` or `ENFORCING`: no-op (return without storing). In ENFORCING mode, deviation detection (Phase 44) handles observations separately.

         - `config = config or {}`.
         - Create `BaselineStore(db_path=db_path)` from `enterprise.behavioral.baseline_store`.
         - Create `BehavioralProfiler(baseline_store=store, config=config)` from `enterprise.behavioral.profiler`.
         - Log `"behavioral_profiling_initialized"` with db_path and default_mode from config.
         - Return the profiler.
       - Follow `enterprise/auth/bootstrap.py` pattern exactly.

    3. Update `enterprise/behavioral/__init__.py`:
       - Keep BSL 1.1 docstring.
       - Add imports for public API: `from .baseline_store import BaselineStore`, `from .profiler import BehavioralProfiler`, `from .bootstrap import bootstrap_behavioral`.
       - Add `__all__` list.
