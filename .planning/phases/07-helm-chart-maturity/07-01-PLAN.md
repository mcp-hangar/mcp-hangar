**Server test** (`packages/helm-charts/mcp-hangar/templates/tests/test-connection.yaml`):
Create a Pod resource that validates the server's health endpoint is reachable through the service:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: "{{ include "mcp-hangar.fullname" . }}-test-connection"
  labels:
    {{- include "mcp-hangar.labels" . | nindent 4 }}
  annotations:
    "helm.sh/hook": test
    "helm.sh/hook-delete-policy": before-hook-creation,hook-succeeded
spec:
  containers:
    - name: wget
      image: busybox:1.37
      command: ['wget']
      args: ['--spider', '--timeout=5', '{{ include "mcp-hangar.fullname" . }}:{{ .Values.service.port }}/health']
  restartPolicy: Never
```

Key points:

- Uses `busybox:1.37` (pinned, lightweight, no RBAC needed)
- `--spider` checks URL without downloading content
- `--timeout=5` prevents hanging
- Targets the service name from `_helpers.tpl` on the configured service port with `/health` path
- `hook-delete-policy: before-hook-creation,hook-succeeded` auto-cleans test pods (Helm v4 removed `--cleanup` flag)

**Operator test** (`packages/helm-charts/mcp-hangar-operator/templates/tests/test-health.yaml`):

- Does NOT use CRD existence test (avoids RBAC complications -- CRD absence causes operator crash-loop which fails health test anyway)
- `hook-delete-policy` for clean test reruns

Ensure `templates/tests/` directories are created for both charts.
