<objective>
Add circuit breaker persistence: implement CircuitBreaker.from_dict() to mirror the existing to_dict(), extend ProviderSnapshot with circuit breaker state fields, and ensure round-trip serialization preserves open/closed state across restarts.

Purpose: Without CB persistence, a tripped circuit breaker resets on restart, allowing cascading failures against known-bad providers. This plan ensures CB state survives process restarts.

Output: CircuitBreaker.from_dict(), extended ProviderSnapshot, unit tests confirming round-trip fidelity.
</objective>

- Reconstruct `CircuitBreakerConfig` from `failure_threshold` and `reset_timeout_s` in dict
- Create new `CircuitBreaker(config=config)`
- Set `cb._state = CircuitState(d.get("state", "closed"))`
- Set `cb._failure_count = d.get("failure_count", 0)`
- Set `cb._opened_at = d.get("opened_at")` (float or None)
- Note: `to_dict()` does NOT currently include `opened_at`. Add `"opened_at": self._opened_at` to the existing `to_dict()` return dict so from_dict can restore it.
- Add `from typing import Any` import if not present.
- Return the configured instance.

In `domain/model/event_sourced_provider.py`, add to `ProviderSnapshot` dataclass:

- `circuit_breaker_state: dict[str, Any] | None = None` (optional field at end, default None for backward compat)

- Add `circuit_breaker_state=d.get("circuit_breaker_state")` to constructor call

The `create_snapshot()` method currently does not capture CB state because CB lives on `ProviderGroup`, not `Provider`. No change needed to `create_snapshot()` -- the `ProviderSnapshot` constructor will default `circuit_breaker_state=None`.
