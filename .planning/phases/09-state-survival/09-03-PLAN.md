**Modify `SagaManager._handle_event()`** in `infrastructure/saga_manager.py` to add the idempotency filter BEFORE `saga.handle()`:

```python
def _handle_event(self, event: DomainEvent) -> None:
    with self._lock:
        sagas = list(self._event_sagas.values())

    event_position = getattr(event, 'global_position', None)

    for saga in sagas:
        if saga.should_handle(event):
            # Idempotency check: skip already-processed events
            if self._saga_state_store is not None and event_position is not None:
                if self._saga_state_store.is_processed(saga.saga_type, event_position):
                    logger.debug(
                        "saga_event_already_processed",
                        saga_type=saga.saga_type,
                        event_position=event_position,
                    )
                    continue

            try:
                commands = saga.handle(event)

                # Checkpoint after successful handling (from Plan 01)
                if self._saga_state_store is not None:
                    try:
                        self._saga_state_store.checkpoint(
                            saga_type=saga.saga_type,
                            saga_id=saga._saga_id,
                            state_data=saga.to_dict(),
                            last_event_position=event_position or 0,
                        )
                        if event_position is not None:
                            self._saga_state_store.mark_processed(
                                saga.saga_type, event_position,
                            )
                    except Exception as e:  # fault-barrier: persistence failure must not break event handling
                        logger.error(
                            "saga_checkpoint_failed",
                            saga_type=saga.saga_type,
                            error=str(e),
                        )

                for command in commands:
                    try:
                        self._command_bus.send(command)
                    except Exception as e:
                        logger.error(...)
            except Exception as e:
                logger.error(...)
```

Key design decisions:

- Idempotency check is BEFORE handle() -- this is the core protection against duplicate commands
- mark_processed() is AFTER successful handle() and checkpoint -- we only mark as processed after the state is safely persisted
- Both are inside the same fault-barrier try/except as checkpoint
- If event has no global_position (live events during normal operation), skip the idempotency check entirely -- idempotency is only needed for replayed events

def init_saga(full_config: dict[str, Any] | None = None) -> None:
    """Initialize all sagas with optional persistence."""
    ctx = get_context()
    saga_manager = get_saga_manager()

    # Create SagaStateStore if SQLite event store is configured

__2. Add helper functions:__

`_create_saga_state_store(config)`: Check `config.get("event_store", {})` for SQLite driver. If driver is "sqlite", create `SQLiteConnectionFactory` with path `data/saga_state.db` (separate from event store per research recommendation), then create `SagaStateStore`. Otherwise return `NullSagaStateStore()`.

`_restore_saga_state(store, saga)`: Call `store.load(saga.saga_type)`. If result is not None, call `saga.from_dict(result["state_data"])`. Log restoration.

`_restore_group_circuit_breakers(store, groups)`: Load CB state from saga store with saga_type="circuit_breaker:{group_id}" for each group. If found, call `CircuitBreaker.from_dict()` and replace the group's `_circuit_breaker`. This uses the saga state store as a general-purpose small-state persistence mechanism (CB state is small, ~5 fields).

To PERSIST group CB state, add a periodic checkpoint. The simplest approach: register an event handler on the event bus that listens for `ProviderDegraded` and `HealthCheckFailed` events, and after handling, checkpoints the group's CB state to the saga store. This can be a simple function registered via `event_bus.subscribe()`.

Actually, a simpler approach that avoids complexity: persist group CB state in `SagaManager._handle_event()` alongside saga state. Since `GroupRebalanceSaga` handles the same events and calls `group.report_failure()` which updates the CB, we can checkpoint group CB state after the rebalance saga handles an event. Add this as part of the checkpoint logic specifically for the "group_rebalance" saga type.

Even simpler and more robust: Add a `save_circuit_breaker_state(group_id, cb_dict)` method to `SagaStateStore` that stores CB state under saga_type="circuit_breaker", saga_id=group_id. Call this from `init_saga()` by subscribing a lightweight event handler to the event bus that periodically (or on relevant events) saves CB state. But this adds complexity.

__Recommended approach (per KISS):__ In `_restore_group_circuit_breakers()`, load CB state. For SAVING CB state, add a `shutdown` hook to `ApplicationContext.shutdown()` that iterates groups and saves their CB state to the saga store. This is sufficient -- CB state only matters across restarts, so saving at shutdown is the minimum viable persistence.

Add a `save_group_circuit_breakers(saga_state_store, groups)` function called from shutdown. Add it to `ApplicationContext` by storing the saga_state_store reference.
