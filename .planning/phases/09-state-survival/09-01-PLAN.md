<objective>
Create the saga persistence foundation: a SagaStateStore backed by SQLite (via existing SQLiteConnectionFactory + MigrationRunner), add to_dict/from_dict serialization to all EventTriggeredSaga subclasses, and integrate checkpoint writes into SagaManager._handle_event().

Purpose: Without saga state persistence, all in-progress recovery and failover state is lost on process restart. This plan builds the persistence layer that PERS-02 (idempotency + bootstrap resume) will wire up.

Output: SagaStateStore class, serializable sagas, SagaManager checkpoint integration, unit tests.
</objective>

**Modify `SagaManager._handle_event()`** in `infrastructure/saga_manager.py`:

After each successful `saga.handle(event)` call, if `self._saga_state_store` is not None, call checkpoint OUTSIDE the lock. The pattern:

```python
def _handle_event(self, event: DomainEvent) -> None:
    with self._lock:
        sagas = list(self._event_sagas.values())

    for saga in sagas:
        if saga.should_handle(event):
            try:
                commands = saga.handle(event)

                # Checkpoint after successful handling (outside lock)
                if self._saga_state_store is not None:
                    try:
                        self._saga_state_store.checkpoint(
                            saga_type=saga.saga_type,
                            saga_id=saga._saga_id,
                            state_data=saga.to_dict(),
                            last_event_position=getattr(event, 'global_position', 0) or 0,
                        )
                    except Exception as e:  # fault-barrier: checkpoint failure must not break event handling
                        logger.error(
                            "saga_checkpoint_failed",
                            saga_type=saga.saga_type,
                            error=str(e),
                        )

                for command in commands:
                    try:
                        self._command_bus.send(command)
                    except Exception as e:  # fault-barrier: command failure must not crash event handler
                        logger.error(...)
            except Exception as e:  # fault-barrier: saga handler failure must not crash event bus
                logger.error(...)
```

Key rules:

- Checkpoint happens AFTER `saga.handle()` but BEFORE command execution (so if commands fail, we still have the state saved)
- Actually, re-reading the research: checkpoint should happen after handle() and the state it captures is the POST-handle state. The commands list is separate. Checkpoint the saga state dict, not the commands.
- `global_position` may not exist on DomainEvent -- use `getattr(event, 'global_position', 0) or 0` safely
- Wrap checkpoint in try/except as a fault barrier -- checkpoint failure must not prevent event handling

**Update existing tests** in `test_saga_manager.py` to cover checkpoint behavior. Add new test cases for the behaviors listed above. Use Mock(spec=SagaStateStore) or the NullSagaStateStore.
