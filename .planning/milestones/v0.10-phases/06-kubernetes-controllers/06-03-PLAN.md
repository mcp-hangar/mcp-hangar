3. Configure envtest.Environment:

    t.Helper()

   - Create MCPProviderGroup with selector `{matchLabels: {app: "web"}}`
   - Create 2 MCPProviders with label `app: web` (one Ready, one Degraded)
   - Create 1 MCPProvider WITHOUT the label (should not be selected)
   - Wait for group status to show ProviderCount=2, ReadyCount=1, DegradedCount=1

2. **TestMCPProviderGroup_StatusAggregation** (K8S-02):
   - Create group with selector matching all test providers
   - Create providers in various states: 2 Ready, 1 Degraded, 1 Dead, 1 Cold
   - Wait for group status
   - Assert: ProviderCount=5, ReadyCount=2, DegradedCount=1, DeadCount=1, ColdCount=1
   - Assert Providers slice has 5 entries with correct states

3. **TestMCPProviderGroup_HealthPolicyThreshold** (K8S-03):
   - Create group with HealthPolicy.MinHealthyPercentage=60
   - Create 3 Ready + 2 Dead providers (60% healthy)
   - Wait for Ready=True (threshold met at exactly 60%)
   - Assert Degraded=True (dead providers exist, coexists with Ready)
   - Assert Available=True (at least 1 ready)

4. **TestMCPProviderGroup_ZeroMembers** (K8S-03):
   - Create group with selector that matches nothing
   - Wait for Ready=Unknown with reason "NoProviders"
   - Assert Available=False, Degraded=False

5. **TestMCPProviderGroup_CoexistingReadyDegraded** (K8S-03):
   - Create group with HealthPolicy.MinHealthyPercentage=30
   - Create 2 Ready + 3 Degraded providers (40% healthy, above 30% threshold)
   - Wait for Ready=True AND Degraded=True simultaneously
   - Assert both conditions are True

6. **TestMCPProviderGroup_Deletion** (cleanup):
   - Create group with providers
   - Delete the group
   - Wait for group to be fully removed (finalizer cleanup)
   - Assert providers still exist (group does not own providers)

**Helper functions:**

```go
func waitForDiscoveryCondition(t *testing.T, name, namespace, condType string, status metav1.ConditionStatus) {
    t.Helper()
    require.Eventually(t, func() bool {
        source := &mcpv1alpha1.MCPDiscoverySource{}
        if err := k8sClient.Get(ctx, types.NamespacedName{Name: name, Namespace: namespace}, source); err != nil {
            return false
        }
        for _, c := range source.Status.Conditions {
            if c.Type == condType && c.Status == status {
                return true
            }
        }
        return false
    }, 10*time.Second, 250*time.Millisecond, "condition %s=%s not met", condType, status)
}

func waitForManagedProviderCount(t *testing.T, sourceName, namespace string, count int) {
    t.Helper()
    require.Eventually(t, func() bool {
        providerList := &mcpv1alpha1.MCPProviderList{}
        if err := k8sClient.List(ctx, providerList,
            client.InNamespace(namespace),
            client.MatchingLabels{LabelDiscoveryManagedBy: sourceName},
        ); err != nil {
            return false
        }
        return len(providerList.Items) == count
    }, 15*time.Second, 250*time.Millisecond, "expected %d managed providers", count)
}
```

**Test cases (each a separate test function):**

1. **TestMCPDiscoverySource_ConfigMapDiscovery** (K8S-04):
   - Create namespace
   - Create ConfigMap with key "providers.yaml" containing YAML:

     ```yaml
     provider-a:
       mode: remote
       endpoint: http://provider-a:8080
     provider-b:
       mode: remote
       endpoint: http://provider-b:8080
     ```

   - Create MCPDiscoverySource with type=ConfigMap, mode=Additive, configMapRef pointing to the ConfigMap
   - Wait for Synced=True condition
   - Assert 2 MCPProvider CRs created in the namespace
   - Assert each has label `mcp-hangar.io/managed-by: <source-name>`
   - Assert each has owner reference pointing to the MCPDiscoverySource
   - Assert DiscoveredCount=2 in status

2. **TestMCPDiscoverySource_AdditiveNeverDeletes** (K8S-05):
   - Create namespace
   - Create ConfigMap with 2 providers
   - Create MCPDiscoverySource with mode=Additive
   - Wait for 2 MCPProviders created
   - Update ConfigMap to have only 1 provider
   - Trigger reconcile by updating source annotation (or wait for requeue)
   - Wait for status to update (DiscoveredCount may change)
   - Assert: 2 MCPProviders still exist (additive never deletes)

3. **TestMCPDiscoverySource_AuthoritativeDeletes** (K8S-05):
   - Create namespace
   - Create ConfigMap with 2 providers
   - Create MCPDiscoverySource with mode=Authoritative
   - Wait for 2 MCPProviders created
   - Update ConfigMap to have only 1 provider (remove provider-b)
   - Trigger reconcile (annotate source or wait for requeue)
   - Wait for managed provider count to drop to 1
   - Assert: Only provider-a exists, provider-b was deleted

4. **TestMCPDiscoverySource_OwnerReferences** (K8S-06):
   - Create ConfigMap discovery source
   - Wait for MCPProviders created
   - Assert each MCPProvider has:
     - OwnerReference with source's Name, UID, Kind="MCPDiscoverySource"
     - Label `mcp-hangar.io/managed-by` set to source name
   - Delete the MCPDiscoverySource
   - Wait for MCPProviders to be garbage collected (owner reference cascade)

5. **TestMCPDiscoverySource_PausedFreeze** (K8S-05):
   - Create ConfigMap source, wait for providers created
   - Update source: set `spec.paused=true`
   - Wait for Paused=True condition
   - Update ConfigMap (add provider-c)
   - Wait 3 seconds (enough for at least one requeue cycle)
   - Assert: NO new provider-c created (paused freezes scans)
   - Update source: set `spec.paused=false`
   - Wait for provider-c to be created (resumes)

6. **TestMCPDiscoverySource_Deletion** (cleanup):
   - Create source with providers
   - Delete the source
   - Wait for finalizer cleanup (source fully removed)
   - Assert managed MCPProviders are cleaned up (either by finalizer or GC)

**Notes on ConfigMap tests:**

- ConfigMap mode is the simplest to test in envtest (no namespace scanning, no pod/service creation needed)
- Focus ConfigMap tests on sync behavior (additive/authoritative/paused) since that tests the core sync engine
- Other discovery modes (Namespace, Annotations, ServiceDiscovery) share the same sync path -- the difference is only the discovery function
