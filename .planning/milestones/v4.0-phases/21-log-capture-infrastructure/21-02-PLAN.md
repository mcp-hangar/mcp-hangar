<objective>
Establish live stderr capture by wiring Provider._start_stderr_reader() — a daemon thread that reads process.stderr line-by-line and appends LogLine entries to the provider's log buffer. Confirms DockerLauncher uses stderr=subprocess.PIPE so the pipe is available.

Purpose: Without this wiring, providers run with stderr either discarded or uncaptured. This is the bridge between running provider processes and the log ring buffer established in Plan 01.
Output: Provider aggregate self-manages its stderr reader thread; launchers expose stderr pipes; unit tests confirm thread lifecycle correctness.
</objective>

class Provider:
    def set_log_buffer(self, buffer: "IProviderLogBuffer") -> None:
        """Inject the log buffer for this provider."""

    def _start_stderr_reader(self, client: Any) -> None:
        """Spawn daemon thread reading client.process.stderr into self._log_buffer.
        No-op if client.process is None or process.stderr is None."""
