# Milestone v7.0: Kubernetes Enforcement Foundation + Licensing

**Status:** COMPLETE
**Target:** 2026-04-15
**Completed:** 2026-03-24
**PyPI release:** v0.13.0
**Phases:** 35-41
**Tracks:** Core (Python), Operator (Go), Helm Charts
**Coordination:** Licensing phases (35-37) are a hard pre-requisite for all other phases and for starting any BSL-licensed work. K8s phases (38-41) can start in parallel once Phase 35 is complete.

## Overview

Two parallel tracks with one strict dependency:

**Licensing track (35-37):** Migrate Pro/Enterprise features from `src/` to `enterprise/` under BSL 1.1, establish the import boundary CI check, and add the license file. This is a structural pre-requisite -- no new enterprise features can be developed until the boundary exists. See `.planning/MIGRATION_ENTERPRISE.md` for the detailed migration table.

**K8s enforcement track (38-41):** Implement the capability declaration schema, NetworkPolicy generation from capabilities in the operator, the enforcement loop (policy reconciliation + violation signals), and admission/policy integration. This makes Hangar the most secure way to run MCP servers in Kubernetes.

MIT licensed work goes in `src/` and `packages/operator/`. BSL licensed work goes in `enterprise/`.
